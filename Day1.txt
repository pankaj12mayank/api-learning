## 🧠 1. What is an API? (Simple Explanation)

API = **A way for apps to talk to each other**

Example:

* Your frontend (React / mobile app) → asks API → gets data

👉 Think like:

```
You → Swiggy app → API → Restaurant database → Response → You
```

## 🧪 3. Create Your First API

Create file:
👉 `main.py`

Paste this:

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def home():
    return {"message": "Hello, API is working 🚀"}

@app.get("/hello")
def say_hello():
    return {"message": "Hello Pankaj 👋"}
```

# 🧠 🔍 Code Breakdown (Deep Understanding)

## ✅ 1. Import FastAPI

```python
from fastapi import FastAPI
```

👉 What this means:

* You are importing the **FastAPI framework**
* This gives you tools to create APIs

📌 Think:

> “I’m bringing a toolkit to build APIs”

---

## ✅ 2. Create App Instance

```python
app = FastAPI()
```

👉 What’s happening:

* You are creating your **API application**

📌 Think:

> “This is my backend server / API system”

Everything you build will attach to this `app`

---

## ✅ 3. First Endpoint (Route)

```python
@app.get("/")
def home():
    return {"message": "Hello, API is working 🚀"}
```

Let’s break it 👇

---

### 🔹 `@app.get("/")`

👉 This is called a **decorator**

Meaning:

* When someone visits `/`
* Using **GET request**
* Run the function below it

📌 Think:

> “If someone hits this URL → run this function”

---

### 🔹 `"/"` (root URL)

* This is your base endpoint
* URL:

```id="hqq9a6"
http://127.0.0.1:8000/
```

---

### 🔹 `def home():`

* This is a normal Python function
* It runs when API is called

---

### 🔹 `return {"message": ...}`

👉 Important:

* You are returning a **Python dictionary**
* FastAPI automatically converts it into **JSON**

📌 Output:

```json
{
  "message": "Hello, API is working 🚀"
}
```

---

# ✅ 4. Second Endpoint

```python
@app.get("/hello")
def say_hello():
    return {"message": "Hello Pankaj 👋"}
```

👉 Same logic, different route:

| URL      | Function    |
| -------- | ----------- |
| `/`      | home()      |
| `/hello` | say_hello() |

---

# 🧠 🏗️ Overall Structure (VERY IMPORTANT)

This pattern is universal:

```id="xzq62w"
Client (browser/app)
        ↓
   URL endpoint (/hello)
        ↓
   API Route (@app.get)
        ↓
   Function runs
        ↓
   Returns JSON response
```

---

# 🌍 Does This Work in Other Languages?

👉 YES — 100% SAME CONCEPT (syntax changes only)

---

## 🔁 Example Comparison

### 🐍 Python (FastAPI)

```python
@app.get("/hello")
def hello():
    return {"msg": "hi"}
```

---

### 🟢 Node.js (Express)

```javascript
app.get("/hello", (req, res) => {
  res.json({ msg: "hi" });
});
```

---

### ☕ Java (Spring Boot)

```java
@GetMapping("/hello")
public String hello() {
    return "hi";
}
```

---

## 🧠 Common Pattern Across ALL APIs

| Concept  | Same Everywhere |
| -------- | --------------- |
| Route    | `/hello`        |
| Method   | GET / POST      |
| Function | Handles request |
| Response | JSON            |

👉 Only syntax changes, **logic stays same**
