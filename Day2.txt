# 🔹 1. Path Parameters (Dynamic URL)

## ✅ Code:

```python
@app.get("/users/{user_id}")
def get_user(user_id: int):
    return {"user_id": user_id}
```

---

## 🔍 What does this mean?

### 🧩 Step-by-step flow:

### 1. Client hits URL:

```bash
/users/5
```

---

### 2. FastAPI sees:

```python
"/users/{user_id}"
```

👉 It understands:

> “Something will come in place of `{user_id}`”

---

### 3. Value Extraction

From:

```bash
/users/5
```

👉 It extracts:

```python
user_id = 5
```

---

### 4. Type Conversion

```python
user_id: int
```

👉 FastAPI converts:

```python
"5" → 5 (integer)
```

If invalid:

```bash
/users/abc ❌
```

👉 Error automatically 🚀

---

### 5. Function Execution

```python
def get_user(user_id: int):
```

👉 Function receives:

```python
user_id = 5
```

---

### 6. Response

```python
return {"user_id": user_id}
```

👉 Output:

```json
{
  "user_id": 5
}
```

---

# 🧠 Key Idea

```text
URL → Extract value → Validate → Pass to function → Return response
```

---

# 🔹 2. Query Parameters (Filtering Data)

## ✅ Code:

```python
@app.get("/search")
def search_user(name: str, age: int):
    return {"name": name, "age": age}
```

---

## 🔍 Flow:

### 1. Client request:

```bash
/search?name=Pankaj&age=28
```

---

### 2. FastAPI reads URL:

Split:

```text
/search        → route
?name=Pankaj   → query param
&age=28        → query param
```

---

### 3. Mapping to function:

```python
def search_user(name: str, age: int)
```

👉 It assigns:

```python
name = "Pankaj"
age = 28
```

---

### 4. Type validation

* `name` → string ✅
* `age` → integer ✅

If wrong:

```bash
/search?name=Pankaj&age=abc ❌
```

👉 Error

---

### 5. Response

```json
{
  "name": "Pankaj",
  "age": 28
}
```

---

# 🧠 Key Idea

```text
URL ?key=value → Extract → Validate → Pass → Return
```

---

# 🔹 3. Combine Both (Very Important)

## ✅ Code:

```python
@app.get("/users/{user_id}/details")
def user_details(user_id: int, active: bool = True):
    return {
        "user_id": user_id,
        "active": active
    }
```

---

## 🔍 Flow:

### Request:

```bash
/users/10/details?active=false
```

---

### FastAPI does:

| Part        | Value          |
| ----------- | -------------- |
| Path param  | user_id = 10   |
| Query param | active = false |

---

### Function receives:

```python
user_id = 10
active = False
```

---

### Output:

```json
{
  "user_id": 10,
  "active": false
}
```

---

# 🔥 4. Default Values (Optional Params)

## ✅ Code:

```python
@app.get("/products")
def get_products(category: str = None):
    return {"category": category}
```

---

## 🔍 Behavior:

### Case 1:

```bash
/products
```

👉 Output:

```json
{"category": null}
```

---

### Case 2:

```bash
/products?category=electronics
```

👉 Output:

```json
{"category": "electronics"}
```

---

# 🧠 Key Idea

```text
If value not provided → default is used
```

# 🌍 Same Concept in Other Languages?

👉 YES — same structure everywhere

---

## 🟢 Node.js (Express)

```javascript
app.get("/users/:id", (req, res) => {
  res.json({ user_id: req.params.id });
});
```

---

## ☕ Java (Spring Boot)

```java
@GetMapping("/users/{id}")
public int getUser(@PathVariable int id) {
    return id;
}
```

---

# 🧠 Universal API Pattern (Important)

```text
1. URL comes in
2. Framework matches route
3. Extract parameters
4. Validate types
5. Call function
6. Return JSON
```

---
