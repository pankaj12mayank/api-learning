# ✅ **DAY 4 – Response Models (Clean & Professional APIs)**

## 🎯 Goal

* Control what your API returns
* Hide unwanted fields
* Structure responses like real production APIs

---

# 🧠 1. Problem Without Response Model

Right now you return:

```python
return user
```

👉 This returns **everything**

❌ Problem:

* You may expose sensitive data later (password, etc.)
* No control over output

---

# 🔥 2. Solution = Response Model

👉 You define what response should look like

---

# 🧩 3. Example (Important)

## ✅ Step 1: Create Input Model

```python
from pydantic import BaseModel

class User(BaseModel):
    name: str
    age: int
```

---

## ✅ Step 2: Create Response Model

```python
class UserResponse(BaseModel):
    name: str
```

👉 Notice:

* No `age`
* Only `name`

---

## ✅ Step 3: Use response_model

```python
from fastapi import FastAPI

app = FastAPI()

@app.post("/users", response_model=UserResponse)
def create_user(user: User):
    return user
```

---

# 🔍 What Happens Internally

### Input:

```json
{
  "name": "Pankaj",
  "age": 28
}
```

---

### Output:

```json
{
  "name": "Pankaj"
}
```

👉 `age` is removed automatically ✅

---

# 🧠 Why This Is Powerful

```text
You separate:
Input Model  ≠  Output Model
```

---

# 🏗️ Real-World Example

### Input (Register user):

```json
{
  "name": "Pankaj",
  "email": "p@test.com",
  "password": "123456"
}
```

---

### Output (Response):

```json
{
  "name": "Pankaj",
  "email": "p@test.com"
}
```

👉 Password hidden 🔒

# 🔥 4. Multiple Response Fields

```python
class UserResponse(BaseModel):
    name: str
    age: int
    message: str
```

---

```python
@app.post("/users", response_model=UserResponse)
def create_user(user: User):
    return {
        "name": user.name,
        "age": user.age,
        "message": "Created successfully"
    }
```

# 🚨 Common Mistakes

### ❌ Returning extra fields

```python
return {"name": user.name, "extra": "data"}
```

👉 FastAPI will remove `extra`

---

# 🧠 Key Concept (VERY IMPORTANT)

```text
response_model = output filter + validation
```
